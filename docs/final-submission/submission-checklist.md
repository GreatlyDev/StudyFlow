# StudyFlow Final Submission Checklist

Use this checklist before submitting on Canvas.

## Group Submission Items

### 1. Source Code, External Libraries, and Dependency Files

Submit one of the following:

- GitHub repository link: `https://github.com/GreatlyDev/StudyFlow`
- A ZIP file of the repository source code if Canvas requires file upload

Make sure the submission includes:

- `backend/`
- `frontend/`
- `docs/`
- `README.md`
- `.gitignore`
- `backend/requirements.txt`
- `frontend/package.json`
- `frontend/package-lock.json`
- `backend/.env.example`

Do not submit:

- `.venv/`
- `node_modules/`
- `backend/.env`
- OpenAI API keys
- local database files unless the professor specifically asks for demo data

### 2. User Documentation

Recommended file:

- `docs/final-submission/user-documentation.md`

This document explains:

- How to install and run the app
- How to configure the backend `.env`
- How to use each feature
- Where screenshots should be inserted

Before submitting, add screenshots from your local app where the document says `[Screenshot to add]`.

### 3. Trello Board Evidence

Recommended file:

- `docs/final-submission/trello-management-report.md`

This document explains:

- How the team used Trello
- Meeting minutes
- How task cards show member responsibilities
- How deadlines were tracked

Recommended screenshots:

- Full Trello board
- Product Backlog
- Sprint Backlog
- To Do / In Progress / Done
- Meetings list
- Example card with description, labels, members, and due date

### 4. Project Report

Recommended file:

- `docs/final-submission/project-report.md`

This document explains:

- Overall development process
- User stories
- How stories were implemented
- How work was tracked
- Difficulties and missing features
- How the team tried to overcome problems

### 5. Peer Review Form

This is an individual submission.

Use:

- `C:\Users\great\Downloads\Peer review .docx`

Fill it out privately. Do not commit it to GitHub unless the professor explicitly says to.

### 6. Overall Project Completion

The project is presentation-ready as a working prototype. It includes:

- Account registration and login
- Course management
- Assignment management
- Study schedules
- Reminders and browser alerts
- Study materials
- Flashcards
- Practice quizzes
- Dashboard
- AI-supported recommendations and flashcard generation foundation
